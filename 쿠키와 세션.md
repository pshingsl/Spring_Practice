# Cookie & Session 정리

---

## 1. Cookie (쿠키)

### 개념

*  쿠키는 **서버가 클라이언트(브라우저)에 저장하는 작은 데이터**
*  HTTP는 Stateless이기 때문에 상태 유지를 위해 사용


**“클라이언트에 저장되는 상태 정보”**

---

### 구성 요소

*  이름(Name)
*  값(Value)
*  만료 시간(Expires / Max-Age)
*  경로(Path)
*  도메인(Domain)

---

### 용량 / 개수 제한

*  브라우저별로 다름 (일반적으로 4KB 제한)
*  도메인당 20개 제한 → 최신 브라우저에서는 다름

* 쿠키 제한은 **브라우저 구현마다 다르다**

---

### 동작 과정

```text id="cookie-flow"
1. 클라이언트 요청
2. 서버 쿠키를 생성 → Set-Cookie 응답
3. 브라우저 저장
4. 이후 요청 시 Cookie 자동 포함
```

*  브라우저가 자동으로 쿠키를 요청에 포함

---

### 사용 예시

* 로그인 유지
* “오늘 하루 보지 않기”
* 사용자 설정 저장

---

### 특징

*  클라이언트에 저장됨
*  서버 자원 사용 없음
*  요청마다 자동 전송됨

---

### 보안 이슈

*  XSS → 쿠키 탈취 가능
*  CSRF → 쿠키 자동 전송 문제

👉 대응

* HttpOnly
* Secure
* SameSite

---

## 2. Session (세션)

### 개념

*  세션은 일정시간 동안 **서버에 사용자 상태를 저장하는 방식**
*  클라이언트는 session-id만 보유

여기서 일정 시간은 방문자가 웹 브라우저를 통해 웹 서버에 접속한 시점부터 웹 브라우저를 종료하여 연결을 끝내는 시점을 말한다.

즉, 방문자가 웹 서버에 접속해 있는 상태를 하나의 단위로 보고 그것을 세션이라고 한다.

**“서버에 저장되는 사용자 상태”**

---

### 동작 과정

```text id="session-flow"
1. 클라이언트 요청
2. 서버 → session 생성
3. session-id 발급
4. session-id를 쿠키에 저장
5. 요청 시 session-id 전달 존재하지 않는다면 생성 후 전달
6. 서버 → 클라잉너트 정보 요청을 처리 후 응답
```

*  세션도 결국 쿠키(session-id)를 사용

---

### 특징

* 웹 서버에 웹 컨테이너의 상태를 유지하기 위한 정보를 저장한다
* 웹 서버의 저장되는 쿠키(=세션 쿠키)
*  서버 메모리 사용
* 클라이언트는 session-id만 보유
*  데이터 저장 제한 없음 (서버 자원 내)

---

### 장점

* 민감 정보 서버에 저장 → 보안 우수
*  데이터 위변조 위험 낮음

---

### 단점

*  서버 자원 사용 증가
*  서버 확장성에 불리 (Stateful)

---

### 라이프사이클

*  브라우저 종료 시 종료 (기본 설정 기준)
*  서버에서 만료 시간 설정 가능

👉 주의

*  “브라우저 종료 = 무조건 세션 종료”는 아님 (설정에 따라 다름)

---

## 3. Cookie vs Session 차이

| 구분     | 쿠키    | 세션       |
| ------ | ----- | -------- |
| 저장 위치  | 클라이언트 | 서버       |
| 속도     | 빠름    | 상대적으로 느림 |
| 보안     | 낮음    | 높음       |
| 데이터 크기 | 제한 있음 | 제한 거의 없음 |
| 서버 부담  | 없음    | 있음       |

---

## 4. 핵심 차이 

 **저장 위치가 아니라 “상태 관리 방식” 차이**

* 쿠키 → Stateless 유지
*  세션 → Stateful 방식

---

## 5. 왜 쿠키를 사용할까?

*  서버 자원 절약
*  빠른 응답 가능
*  세션만 사용하면 서버 부담 증가

 결론
**쿠키 + 세션 혼합 사용이 일반적**

---

## 6. Cache (캐시)와의 차이

### Cache 개념

*  캐시는 **성능 향상을 위한 임시 저장소**
*  인증과는 무관

---

### 비교

| 구분     | 쿠키/세션      | 캐시      |
| ------ | ---------- | ------- |
| 목적     | 인증 / 상태 유지 | 성능 개선   |
| 저장 데이터 | 사용자 정보     | 리소스 데이터 |
| 위치     | 클라/서버      | 브라우저/서버 |

---

### 캐시 특징

*  이미지, CSS, JS 저장
*  재요청 시 빠른 로딩

---

### 용어

*  Cache Hit → 캐시 사용
*  Cache Miss → 서버 요청

---

## 7. 핵심 정리

*  쿠키 → 클라이언트 저장
* 세션 → 서버 저장
*  세션도 쿠키를 사용함
*  캐시는 성능 최적화용
